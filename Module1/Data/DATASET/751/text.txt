	
В рамках проекта «Современная цифровая образовательная среда в РФ» на базе кафедры
«Физическая культура» и кафедры «Прикладная математика и компьютерная безопасность» Сибирского федерального университета разработано веб-приложение, которое реализует взаимодействие
между спортсменом и тренером в соответствии с современными требованиями информационной
безопасности.
Сам сайт представляет собой систему обмена информации о заданиях. Пользователь «тренер»
создает задание для его последующего выполнения и отчетности со стороны пользователя «спортсмена». Помимо создания заданий «тренер» также может отслеживать процесс их выполнения, а
«спортсмен» вносить данные о выполненных заданиях.
При реализации данного проекты были использованы следующие программные средства:
– редактор кода – Visual Studio Code. Нами было принято решение остановиться на
VisualStudioCode от компании Microsoft, так как данный редактор кода является очень многофункциональным и отлично подходит для веб-разработки, а также не взымает плату за использование.
– среда выполнения программного кода – Nuxt.js. Это фреймворк высокого уровня, который позволяет быстро и относительно легко создавать действительно сложные приложения. Он отвечает
всем современным требованиям безопасности эксплуатации и поддерживает разработку и тестирование приложений прямо во время их выполнения. Nuxt.js позволяет создавать готовые к работе вебприложения, обеспечивает отличную файловую структуру, улучшает роутинг, обеспечивает рендеринг на стороне сервера и позволяет создавать универсальные сервисы [1].
– сервис для хранения данных – Firebase. После изучения доступных сервисов для долговременного хранения данных было принято решение воспользоваться Firebase. Это мобильная платформа Google, которая предоставляет возможность бесплатно создать базу данных для взаимодействия с веб-приложением. Благодаря Firebase можно легко протестировать работоспособность проекта, так как в ней реализовано API для приложений на Nuxt.js.
Весь функционал приложения был реализован с использованием HTML, CSS и JavaScript, где
включены следующие функции:
– страница входа для пользователя,
– регистрация и редактирование данных,
– определение задания от лица тренера,
– внесение результатов выполненных заданий от лица спортсмена,
– организация доступа к введенной ранее информации.
Для каждой из поставленных задач функционала была создана отдельная страница [2].
Концепция подобного приложения предполагает обмен данными между спортсменом и закрепленным за ним тренером. Тренер может создавать задания и следить за результатами их выполнения. Спортсмен же имеет право предоставлять данные о выполненных заданиях. Все данные хранятся в защищенной базе данных с непрерывным доступом. Пользователь может ознакомиться со справочной информацией, правилами сообщества и политикой конфиденциальности в отдельных вкладках. Если пользователь уже зарегистрирован, то сможет сразу войти в приложение и попасть на глав-

ную страницу. С главной страницы можно перейти на одну из двух страниц, реализующих весь необходимый функционал.
Страница аутентификации приложения содержит два поля для ввода данных, необходимых для
входа в приложение: электронный адрес и пароль. Пароль удовлетворяет современным требованиям
безопасности. Электронный адрес, введенный при регистрации в системе, проходит процедуру подтверждения: на него отправляется специальный код.
После аутентификации пользователь попадает на главную страницу, на которой находится
подменю. Для спортсмена этими пунктами будут «Профиль» и «Пароль». Для тренера добавятся ещё
два пункта: «Спортсмены» и «Поиск». Подобная реализация призвана упростить взаимодействие между пользователем и системой и сделать его интуитивно понятным [3].
Функционал вкладок:
– «Спортсмены» – список спортсменов, закрепленных за авторизовавшимся тренером.
– «Поиск» – форма для добавления новых спортсменов в список тренера.
– «Профиль» – форма для изменения введенных ранее конфиденциальных данных.
– «Пароль» – форма для изменения введенного ранее пароля.
Помимо «Главной страницы» создана отдельная вкладка «Задания». На ней размещен основной функционал приложения. Перейдя в нее, спортсмен сможет получить возможность просмотреть
свои задания, а тренер выдать новые (рисунок 1).
