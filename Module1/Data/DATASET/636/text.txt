Процесс создания сайтов и веб-приложений, который состоит из
множества этапов, которые непосредственно взаимосвязаны между собой,
называется веб-разработка. Для понимания принципа еѐ работы, необходимо
разобрать из каких частей состоит сайт.
Каждый сайт включает в себя пользовательскую и серверную часть
интерфейса. Любая страница в интернете имеет текст, кнопки, панели,
изображения и видео, по которому есть возможность перемещаться,
свободно изучать контент. Нам представлена пользовательская часть сайта и
результат труда frontend-разработчика: визуализация, интерактивность и
понятность интерфейса.
За логику, работоспособность и точное функционирование сайта несет
ответственность серверная часть, которая скрыта от пользователя. Еѐ
формированием занимается backend-разработчик, а регулировать данный
процесс может только администратор сайта через специальный интерфейс.
Запрос, который осуществляет пользователь, передается на сервер,
который основную работу совершает там. Данный запрос обрабатывается,
фильтруется, а ответ отправляется обратно. Backend-разработка отвечает за
правильное выполнение этого процесса. В этом процессе пользователю
доступна только та часть, которая не влияет на его работу, т.е. пользоваться
продуктом, но не имеете доступа к коду.
Серверная и пользовательская часть взаимосвязаны между собой и
предоставляют хороший результат, доступный и функциональный сайт при
слаженной работе.
Взаимодействие frontend- и backend-разработки схоже на
согласованную работу человеческого тела и нервной системы. Подкожные
рецепторы посылают информацию в мозг, а он обрабатывает полученные
данные и отправляет ответный импульс, заставляя тело реагировать. Frontend
ответственен за то, как выглядит продукт, Backend — за то, как этот данный
продукт функционирует.
Теперь наглядно просмотрим процесс общения клиентов и серверов
или как это работает на языке компьютера.
Имеются несколько клиентов. Один из клиентов — браузер
компьютера, которому необходимо получить информацию из интернета.
Производит запрос: вводите вопрос в поисковик Yandex или Google и
одновременно открывается страница с нужной информацией.
Каково процесс работы внутри системы на самом деле? Клиентбраузер, отправляет запрос на сервер. Изначально на сервер пользователя
frontend.
Frontend-сервер (компьютер) обрабатывает запрос, следом выбирает
backend -сервер, который в данный момент свободен, и отправляет запрос из
браузера. Backend-сервер обрабатывает запрос, обращается к базе данных и
посылает ответ на запрос обратно fronted-серверу. А frontend, который
отвечает за удобство пользователя, уже отображает ответ на запрос в виде
HTML-страницы.
Основная задача frontend разработчика— создать доступный и
понятный интерфейс, с которым будет легко работать. Frontend — это не
только дизайн, но и код, который помогает взаимодействовать с
пользователем.
За содержание сайта отвечает язык гипертекстовой разметки HTML—
не представляет собой язык программирования, но данный язык помогает
заполнить сайт необходимой информацией и расположить ее в нужных
частях страницы. Все, что увидите на сайте, — это HTML-файл. Но просто
HTML-страница без оформления выглядит уныло и непривлекательно.
Поэтому, чтобы оформить сайт, сделать его приятным для восприятия, нужен
инструментарий CSS. Самые широкие возможности, включая адаптивности
дизайна и анимацию, предоставляет версия HTML5, на которую сегодня и
стоит ориентироваться.
CSS — оформление сайта, который является языком и отвечает за
отображение HTML-документа. С его помощью верстка визуальной части
сайта упрощается, при этом появляется определенный стиль. Для того, чтобы
добавить интерактивности и динамики, например, мигающие кнопки, можно
использовать JavaScript — интерактивность сайта, это полноценный язык
программирования, который в веб-разработке используется для оживления
сайта. JQuery — Фреймворк языка JavaScript, библиотека с набором готовых
функций, которые упрощают написание JavaScript-кода. Фреймворк
позволяет не писать код заново, а подобрать подходящую часть из готового
набора.
Backend — это процесс объединения сервера с пользователем, и для
создания серверной части сайта нужно освоить полноценный язык
программирования. Backend-разработчик может использовать любые
инструменты, доступные на его сервере, вправе выбрать любой из
универсальных языков программирования, например, Ruby, PHP, Python,
Java. Также для backend-разработки используются разные системы
управления базами данных: MySQL, PostSQL, SQLite, MongoDB.
Таким образом, современная веб-разработка — это непростой и
многоуровневый процесс, который разделяется на две части:
пользовательскую (клиентскую) и серверную. В арсенале Frontend и Backend
имеется масса инструментов, выбор которых зависит от поставленных задач
и сложности проекта. Оба вида разработки включают в себя технические и
творческие компоненты.
