Процесс преобразования ресурсов (ППР) является инструментом
формализации процессов организационно-технических систем и представляет собой преобразователь входных ресурсов в выходные при условии
наличия свободных средств выполнения и срабатывании условий запуска
процесса. К процессам организационно-технических систем относятся такие классы процессов, как производственные, логистические и организационные (бизнес-) процессы. Мультиагентная модель процесса преобразования ресурсов (МППР) разработана на основе интеграции имитационного, экспертного, ситуационного и мультиагентного моделирования [1].
Параметрический синтез ППР предназначен для выработки управляющих
воздействий на систему с целью повышения эффективности ее работы.
Параметрический синтез процесса с использованием модели МППР предполагает проведение экспериментов с моделью согласно плану экспериментов и выбор в дальнейшем эксперимента с лучшим результатом. К недостаткам данного подхода следует отнести необходимость формирования такого плана экспериментов, который содержал бы в себе эффективное
решение, что в случае полнофакторного эксперимента достаточно ресурсоемко в части машинного времени. Актуальным является применение
гибридных эвристико-имитационных алгоритмов, уменьшающих пространство поиска, для решения задачи параметрического синтеза ППР.
В работе рассмотрено применение разработанного эволюционноимитационного алгоритма [2] для решения задачи параметрического синтеза ППР. Эволюционно-имитационный алгоритм программно реализован
в модуле оптимизации процессов предприятия автоматизированной системы выпуска металлургической продукции [3]. 
Рассмотрим задачу параметрического синтеза процесса сопровождения телекоммуникационных систем цехов прокатки металлургического
предприятия. Исследуемый процесс включает в себя прием, обработку и
разрешение трех типов инцидентов (сервисные инциденты, запросы на
обслуживание и запросы на консультацию) по шести видам предоставляемого сервиса (IP-телефония, локальная вычислительная сеть, промышленная громкоговорящая связь, промышленное видеонаблюдение, радиосвязь
и телефонная связь) для двух цехов горячей (ГП) и холодной (ХП) прокатки. Исполнителями процесса являются сотрудники отделов сопровождения телекоммуникационных систем цехов; каждый отдел включает в себя
две группы: группу обслуживания оборудования (ГОО) и группу обслуживания телекоммуникационных систем и АСУ ТП (группа ГОТС). Задача параметрического синтеза исследуемого процесса заключается в определении минимального числа сотрудников отделов при сохранении всех
должностей и при обеспечении максимальной средней загрузки каждого
сотрудника и максимального процента разрешенных инцидентов. 
На основании собранных статистических данных функционирования
исследуемого процесса была разработана мультиагентная модель процесса
преобразования ресурсов. Агенты в модели используются для распределения инцидентов по типам и видам внутри отделов в зависимости от квалификации специалистов и наличия у них свободного времени на разрешение инцидентов. В качестве управляемого (синтезируемого) параметра
процесса выступает количество сотрудников с разбивкой по должностям в
каждом отделе системы.
Работа эволюционно-имитационного алгоритма подразумевает динамическое изменение пространства поиска решения (формирование новой популяции в терминах генетического алгоритма [4]) с учетом оценки
эффективности решений текущего пространства поиска (функций пригодности в терминах генетического алгоритма) и степени оригинальности
решений (меры новизны в терминах алгоритма поиска новизны [5]). Оценка эффективности решения осуществляется с помощью модели МППР, на
вход которой подаются значения управляемых (синтезируемых) параметров, а на выходе формируется значение целевой функции решения.
Для работы эволюционно-имитационного алгоритма первоначально
с помощью проведения экспериментов с моделью процесса были выделены следующие управляемые параметры процесса. Для цеха горячей прокатки: число инженеров в отделе ГОО (4 чел.), число ведущих инженеров
(2 чел.) и администраторов (4 чел.) в отделе ГОТС; для цеха холодной
прокатки: число ведущих инженеров (1 чел.) и администраторов (4 чел.) в
отделе ГОТС. В скобках указано количество сотрудников в отделах для
базового эксперимента «как есть».
Результаты применения эволюционно-имитационного алгоритма к
решению поставленной задачи приведены на рис. 1 в виде фрагмента окна
работы пользователя с модулем оптимизации процессов автоматизированной системы выпуска металлургической продукции. В ходе работы алгоритма проводилась максимизация выходного параметра исследуемого
процесса «Средняя загрузка всех сотрудников в процентах». Найденное
решение, представленное на рис. 1 в столбце «Оптимальное значение»,
соответствует значению средней загрузки всех сотрудников, равной 62%.
На рис. 2 приведен пример изменения для базового эксперимента
средней загруженности в процентах инженеров группы обслуживания
оборудования в цехе горячей прокатки в течение месяца (14480 минут), а
на рис. 3 — пример изменения средней загруженности инженеров той же
группы за тот же период для эксперимента с лучшим результатом.
Снижение числа инженеров, ответственных за разрешение определенных типов инцидентов, позволило повысить среднюю загруженность
группы инженеров из отдела ГОО цеха горячей прокатки на 13,6%, при
этом процент разрешенных инцидентов на конец месяца остался высоким
(выше 90% для всех инцидентов). Также снижение числа ведущих инженеров из отдела ГОТС цеха холодной прокатки позволило повысить среднюю загруженность группы инженеров цеха холодной прокатки на 34%.
В докладе рассмотрено применение эволюционно-имитационного
алгоритма для решения задачи параметрического синтеза процесса преобразования ресурсов. Интеграция эволюционного и имитационного моделирования при решении данной задачи позволяет сократить пространство
поиска решения (количество проведенных экспериментов с моделью для
нахождения эффективного/оптимального решения). При решении задачи
определения эффективного числа сотрудников отделов телекоммуникационного сопровождения цехов прокатки выработаны практические рекомендации, обеспечивающие максимизацию средней загрузки сотрудников
и процента разрешенных инцидентов.
